AI Workflow Journey

Building Production Software with AI

A transformation story: from year-long timelines to week-long deliveries

Phase 01 — Discovery

The "Oh Shit" Moment

Traditional software development timelines were fundamentally changing, and I almost missed it. For years, I'd said it took about a year to build anything complex for clients. That timeline felt immutable.

The catalyst was deceptively simple: a voice-activated kitchen timer built in under an hour. Then came the real revelation—working with my 11-year-old son to build a D&D tracker. We had something genuinely useful in under an hour.

Dungeon Tracker App

Phase 02 — Experimentation

Tool Discovery & Evolution

Started with Replit for rapid prototyping, moved to Cursor for better file control, explored various AI coding assistants. Each tool taught lessons about what works and what doesn't.

The breakthrough came with Claude Code running in the terminal inside VS Code—giving access to all files and CLI tools while maintaining power and speed.

Claude Code in Terminal
Multiple windows, parallel agents
Complete file & CLI access

Phase 03 — Refinement

The Methodology Emerges

TypeScript became non-negotiable—the forced typing prevents entire categories of AI-generated errors. Single-page applications work better because they're easier for AI to maintain context.

"Research, Plan, Build, Test. When you have a problem, have the AI research first, then plan with clear success criteria, then execute in phases."
