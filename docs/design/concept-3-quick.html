Virgin America Digital Transformation ‚Äî In 2013, I led the UX strategy and interaction design for Virgin America's groundbreaking responsive website, the first in the airline industry.

Working closely with Virgin America's in-house team, we tackled the challenge of creating a seamless booking experience across all devices when mobile airline booking was considered nearly impossible.

02
Problem & Challenges
Mobile traffic surging but booking flows desktop-only...
‚ñº

Mobile traffic was growing rapidly, but users had to pinch, zoom, and struggle through tiny forms. The booking flow included 7+ screens and complex interactions like seat selection with 150+ elements.

Key Challenges:

  • Complex multi-step booking flow with form validation
  • Interactive seat map requiring precise touch targets
  • Payment security concerns on mobile browsers
  • Stakeholder skepticism about mobile booking viability
03
Design Approach
Mobile-first wireframes with touch-friendly interactions...
‚ñº

We started mobile-first, reimagining every interaction for touch. The seat selection interface was completely redesigned as a scrollable, zoomable map with clear visual hierarchy instead of trying to shrink the desktop experience.

Every tap target was tested for ergonomic comfort. We used progressive disclosure to reduce cognitive load, showing users only what they needed at each step.

04
Technical Implementation
Responsive CSS Grid, SVG seat maps, progressive enhancement...
‚ñº

Built with responsive CSS Grid (cutting-edge for 2013), optimized JavaScript for touch events, and progressive enhancement for older mobile browsers.

The seat map used SVG for crisp rendering at any zoom level, with touch gesture support for pan and zoom. Payment integration leveraged device-native keyboards for optimal data entry.

Performance: Sub-2-second load on 3G, 60fps interactions, graceful degradation.

Concept Analysis

Strategic Approach

This concept prioritizes speed and scannability above all else. Every feature is designed for users who are evaluating many candidates and need to assess quickly. Low cognitive load, familiar patterns, progressive disclosure.

Strengths

  • Fastest to Scan: Collapsed sections let users see overview at a glance
  • Lowest Friction: No modals, no page navigation ‚Äî everything expands inline
  • Space Efficient: Works perfectly on mobile with minimal scrolling
  • Clear Hierarchy: Metrics and key info always visible, details on demand
  • Easy to Share: URLs can link to expanded sections (great for recruiters)
  • Simple Implementation: Basic accordion/dropdown patterns, no complex animations

Tradeoffs

  • Less Engaging: Prioritizes function over visual impact
  • Feels Utilitarian: More resume than portfolio ‚Äî lacks "wow factor"
  • Requires Good Headers: Collapsed sections need compelling preview text
  • Demo Feels Hidden: "View Demo" button could be overlooked
  • Less Personality: Straightforward approach doesn't convey creative flair

Best Use Cases

  • HR Recruiters: Non-technical people who need clear, scannable information
  • Busy Decision-Makers: CTOs with limited time, comparing many candidates
  • Mobile Users: Professionals reviewing portfolios on phone during commute
  • High-Volume Screening: When portfolio is one of 50+ being evaluated

Implementation Notes

  • Use CSS max-height transitions for smooth accordion animations
  • Add URL hash support (#section-1) for deep linking to expanded sections
  • Store demo type preference in sessionStorage for consistency
  • Use click-outside handler to close filter dropdown
  • Add keyboard support (Enter to expand, Escape to collapse)
  • Consider adding "Expand All" state persistence
  • Optimize for accessibility ‚Äî ARIA labels, focus management
Design Principle: Progressive Disclosure Over Information Density
This concept embraces the idea that not everyone needs to see everything. Give people the overview, let them choose what to explore. This reduces cognitive load and makes assessment faster for busy people while still allowing deep dives when needed.
