01 // Overview

Virgin America: First Responsive Airline Website

In 2013, I led the UX design and front-end architecture for Virgin America's groundbreaking responsive website redesign.

At the time, no major airline had successfully implemented a mobile-first booking experience. We changed that.

02 // Challenge

Airlines were notoriously slow to adopt modern web standards. Most booking flows required desktop computers and frustrated mobile users.

Conversion rates hovered below 2%, and mobile traffic was growing 40% year-over-year with no mobile solution in sight.

Virgin America wanted to break industry patterns and create a delightful mobile booking experience.

03 // Solution

We designed a mobile-first responsive system that adapted seamlessly from 320px phones to 2560px desktops.

Key innovations included progressive disclosure for complex forms, gesture-based seat selection, and real-time price updates that worked offline.

The entire booking flow could be completed on a smartphone with zero friction.

The redesign won multiple industry awards including Webby Awards and became a case study taught at Stanford and MIT.

05 // Learnings

Mobile-first isn't just smaller desktop. It requires rethinking information architecture from the ground up.

Progressive disclosure works. Breaking complex flows into manageable steps dramatically improved completion rates.

Test early, test often. We ran usability tests every two weeks with real travelers at SFO airport.

Concept 2 Analysis

✓ Strengths

  • Bold, distinctive aesthetic that stands out from typical portfolios
  • Drawer panel is non-blocking - users can dismiss easily to keep browsing
  • Horizontal scroll creates immersive, cinematic storytelling experience
  • Compact grid shows more projects above the fold
  • Technical aesthetic resonates with developer/engineer audiences

⚠ Tradeoffs

  • Brutalist aesthetic may feel too aggressive for some audiences
  • Horizontal scroll is less intuitive than vertical (requires user education)
  • Drawer can feel disconnected from main flow if users don't realize it overlays
  • Monospace fonts sacrifice some readability for aesthetic
  • Not accessible for users who rely on standard scroll patterns
